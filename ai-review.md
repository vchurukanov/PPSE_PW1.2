## Проблема 1: [Callback hell та слабка масштабованість]
**Знайдено AI**: "Функції `getData/postData` залежать від callback API, що ускладнює композицію запитів і централізовану обробку помилок."  
**Моя оцінка**: Погоджуюсь. Для складніших сценаріїв (ланцюжки запитів, retry, таймаути) callbacks дуже незручні.  
**Дія**: Виправлено

## Проблема 2: [Дублювання логіки між getData/postData]
**Знайдено AI**: "Обробка статусу, JSON.parse та error-string дублюється у двох функціях."  
**Моя оцінка**: Погоджуюсь. Дублювання вже призводить до розбіжностей (наприклад, різна обробка подій помилок).  
**Дія**: Виправлено

## Проблема 3: [Відсутність retry-логіки]
**Знайдено AI**: "При тимчасових мережевих збоях запит одразу падає без повторної спроби."  
**Моя оцінка**: Погоджуюсь. Для API-клієнта це критичний недолік на нестабільній мережі.  
**Дія**: Виправлено

## Проблема 4: [Сирі рядки помилок замість типізованих Error]
**Знайдено AI**: "Повернення `cb('Error: ...')` ускладнює обробку на верхньому рівні."  
**Моя оцінка**: Погоджуюсь. Типізована помилка з `status/url/details` значно краща для дебагу.  
**Дія**: Виправлено

## Проблема 5: [Невизначені типи параметрів і результатів]
**Знайдено AI**: "В legacy-коді відсутня типізація payload/response, що збільшує ризик runtime-помилок."  
**Моя оцінка**: Погоджуюсь. Це одна з ключових вимог лабораторної.  
**Дія**: Виправлено

## Проблема 6: [Немає захисту від невалідного JSON]
**Знайдено AI**: "`JSON.parse` викликається без try/catch; некоректна відповідь сервера зламає логіку."  
**Моя оцінка**: Погоджуюсь. Потрібен контрольований throw з діагностикою.  
**Дія**: Виправлено

## Проблема 7: [Немає тестів на edge-cases]
**Знайдено AI**: "Без тестів неможливо гарантувати коректність retry і обробки помилок."  
**Моя оцінка**: Погоджуюсь. Додав тести на 4xx/5xx, мережеві збої, невалідний JSON і payload для POST.  
**Дія**: Виправлено

## Проблема 8: [XHR-залежність замість fetch API]
**Знайдено AI**: "XMLHttpRequest є застарілим підходом для нового коду."  
**Моя оцінка**: Частково погоджуюсь. XHR ще може працювати, але для сучасного коду `fetch` простіше й чистіше.  
**Дія**: Виправлено
